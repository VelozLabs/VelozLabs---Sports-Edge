Graphs & visuals

Four load-bearing charts, each answering one question. Thin marks, a recessive grid, an emphasized endpoint, and text in ink — never in the series color.

Also in the kit

Edge distribution histogram (spot the fat tail) · Tier distribution stacked ordinal bar · Coverage heatmap source_root × date, sequential blue (light = missing) · KPI sparklines on every tile. All share the same grid weight, endpoint emphasis, and ink-colored text. Every line/area chart ships a crosshair tooltip; every bar/cell a per-mark hover.

05

Rules the interface enforces

The model's non-negotiables are also UI contracts. The design makes each one visible.

Every number is a probability until a price is attached. No EV without a price.
01
Price before tiering. An un-priced prop shows the hatched unpriced state — a probability with no tier, never a fake edge.
02
Sign, then color. +EV / −EV always carry a + / − and a ▲/▼. Red-green never carries meaning alone (CVD, print, greyscale).
03
Environment ≠ scoring. Layer charts facet the two groups; you can never read a park signal as if it were a bat signal.
04
Confirmed actives only. A projected-slot badge keeps a pick at L1 until the lineup posts — information error stays separable from judgment error.
05
Count independent roots. Source chips collapse collisions (Statcast-derived feeds → one root) so corroboration is never double-counted.
06
Numbers are monospace, always. Tabular figures so odds, probs, and units align down every column — the ledger discipline, made visual.
